2 Data Model · نموذج البيانات

📦 pending_uploads (جديد أو موسَّع)

  • id uuid PK
  • uploaded_by uuid FK
  • storage_path text
  • organ text
  • modality text
  • plane text
  • sequence text?
  • structures text[]
  • prompt_used text?
  • source 'manual' | 'ai'
  • status pending / approved / rejected
↓ approve

🖼️ atlas_images (موجود)

  • id uuid PK
  • storage_path text · bucket atlas
  • organ_id FK
  • modality text
  • plane text
  • sequence text?
  • structures text[]
  • approved_by uuid
  • approved_at timestamptz
→ links

📖 anatomical_structures (يتوسّع)

  • id uuid PK
  • organ_id FK
  • name_en text
  • name_ar text
  • radlex_id text?
  • ta2_id text?
  • importance 1-5
  • image_count int (computed)
↔ many-to-many via structures[]
3 Build Checklist · قائمة التنفيذ

✨ يبنى جديداً NEW

  1. bulk-upload.html — صفحة الرفع الجماعية (drag & drop + form مصغّر لكل صورة).
  2. جدول pending_uploads (إن لم يكفِ review_queue الحالي).
  3. دالة SQL: upload_atlas_image() — تستقبل file + metadata، تُخزّن في bucket مؤقت، تُنشئ صف pending.
  4. دالة SQL: approve_pending_upload() — تنقل الصورة من bucket المؤقت إلى atlas، تُنشئ صف atlas_images، توسّع anatomical_structures للبنى الجديدة.
  5. Autocomplete component — لاختيار البنى التشريحية من القاموس + «Add new».

♻️ يُعاد استخدامه EXISTS

  • review.html — Review Queue الحالية (تعديل بسيط لدعم source='manual').
  • atlas.html — Atlas Viewer (يقرأ من atlas_images تلقائياً).
  • comparison.html — Comparison panes (يعمل مباشرة).
  • dictionary.html — القاموس (يُشتقّ من anatomical_structures).
  • Supabase Auth + RLS — الصلاحيات محفوظة.
  • bucket atlas — Storage جاهز.
  • Notifications system — يشتغل تلقائياً عند approve/reject.
4 Access · التحكّم بالوصول (رابط مؤقت)

🔒 المرحلة الأولى — مخفية

  • • الصفحة تحت pages/bulk-upload.html
  • • غير موجودة في navbar (accessible via direct URL)
  • • RLS: admin + contributor فقط
  • • توصية: URL هاش مؤقت (?token=xyz) إن أردت تشاركها

🚀 بعد اكتمال الأطلس

  • • (أ) نبقيها للتوسّع المستقبلي — موصى به
  • • (ب) نُخفيها تماماً (نحذف من routing)
  • • (ج) نحوّلها لصفحة عامة للـ contributors
ملاحظة: كل شيء «موجود مسبقاً» في المخطّط يعمل الآن — لا نبني من الصفر. الجديد المطلوب فعلاً هو صفحة الرفع الجماعية + دالتان في SQL. الباقي = تعديلات دقيقة على الصفحات الحالية.
